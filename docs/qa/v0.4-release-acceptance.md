# Slidr Free v0.4 轻量验收清单

本清单适用于没有 Apple 开发者账号的个人开源项目。v0.4 的交付物是源码和明确标注为开发用途的 ad-hoc 应用包，不要求 Developer ID 签名、Apple 公证、多台 Mac 或外接触控板才能放行。

## 1. 自动化验收（必须）

```bash
swift run SlidrFreeCoreChecks
swift test
bash scripts/package-release.sh
bash scripts/test-verify-release-signature.sh
```

预期结果：

- 核心检查和测试全部通过；
- 生成 `release/Slidr-Free.app` 与 `release/Slidr-Free.app.zip`；
- 松散应用和压缩包内应用的版本为 `0.4.0`、构建号为 `4000`、Bundle ID 为 `com.slidr.free`；
- 根目录与应用包内许可证一致；
- 两个应用包均为有效 ad-hoc 签名，验证器会拒绝非 ad-hoc 签名。

## 2. 当前 Mac 冒烟测试（发布前建议）

仅在不会干扰日常环境时授予辅助功能权限。无法进行真实权限或硬件测试时，自动化测试仍是 v0.4 的放行依据，并在发布说明中保留限制。

- [ ] 全新偏好下出现四步首次向导，选择“稍后设置”不会启用应用。
- [ ] 概览页准确显示权限、框架、默认物理设备和监听状态，并给出一个推荐行动。
- [ ] 左、右、上边缘可以直接映射动作；没有任何映射时显示“尚未配置手势”。
- [ ] 15 秒安全测试只识别手势，不改变音量、亮度或浏览器标签页。
- [ ] 中键测试只识别选定手指数的轻点，不发送真实中键事件。
- [ ] 权限撤销、关闭测试、关闭窗口、睡眠或退出应用后没有延迟动作。
- [ ] 菜单栏区分正常、暂停和需要注意三类状态，并提供对应入口。
- [ ] 诊断预览不含用户名、完整路径、当前应用、输入内容或触摸坐标。
- [ ] 英文和简体中文界面中的主要控件可达，状态不只依赖颜色表达。

## 3. 升级与恢复（必须由自动化覆盖）

- v0.3 的边缘开关组合迁移到直接映射后保持等价行为；
- 已有用户不会被强制进入首次向导；
- 设置数据损坏会回退到安全默认值并显示可恢复提示；
- ad-hoc 重建导致 TCC 记录失效时，README 和界面提供退出、移除旧条目、重新添加当前应用的说明。

## 4. 社区验证（非阻塞）

有额外设备或可访问性测试资源时，可补充以下结果，但它们不阻塞个人开源版本：

- 其他受支持 macOS 版本；
- 外接 Apple 触控板以及多输入设备组合；
- VoiceOver 完整任务流、较大系统文字与高倍率缩放；
- 不同 Mac 架构和机型。

发现问题时记录系统版本、设备类型、应用版本、复现步骤和已脱敏诊断摘要，不宣称当前实现支持多设备识别或路由。

## 5. 测试实例清理（必须）

每次启动应用进行测试后，在结束时执行：

```bash
pgrep -fl SlidrFreeApp
```

预期无输出。若仍有实例，先正常退出；必要时终止测试进程，然后再次确认无输出。不得让验收实例继续驻留菜单栏。
