# Simplify Release, Settings, and Cursor Behavior Design

## Problem

After the media-key step gesture work, several parts of Slidr-Free are stale or misleading:

1. The GitHub Release ships a prebuilt `Slidr-Free.app.zip`. The project is ad-hoc signed and not notarized, so a binary asset adds friction (Gatekeeper "damaged" warnings) without much value. The user wants the release to ship source code only.
2. The Settings window still exposes legacy continuous-control options (`Sensitivity`, `Normal step`, `Fine step`, `Fine control`, `Continuous window`, `Cursor freeze`, `Bottom quarter only`) that no longer match the discrete step gesture model and confuse users.
3. The menu bar dropdown has a standalone `Permissions…` item that duplicates the permission status already visible in Settings and Debug.
4. The codebase carries unused cursor-freeze infrastructure (`freezeCursor` / `unfreezeCursor` / `CGWarpMouseCursorPosition`) and a `cursorFreeze` setting. Even though it is currently uncalled, its presence is misleading and risks accidental cursor movement during edge gestures.

## Goals

- GitHub Release `v0.1.0` contains only source code; the prebuilt zip asset is removed and no new zip is uploaded.
- Settings window shows only options relevant to the current discrete-step gesture model.
- Menu bar dropdown no longer has a `Permissions…` item.
- No code path warps, hides, or associates the cursor during volume/brightness edge gestures.
- Codebase no longer carries dead cursor-freeze code or settings.

## Non-Goals

- Do not change gesture recognition, media-key posting, or physical trackpad monitor behavior.
- Do not add new features.
- Do not change Debug panel contents beyond removing permission-only menu plumbing if needed.
- Do not introduce notarization or Developer ID signing.

## Chosen Approach

### Release: Source Code Only

- Delete the existing `Slidr-Free.app.zip` asset from GitHub Release `v0.1.0` via `gh release delete-asset`.
- Do not upload any new binary asset for this or future releases in this iteration.
- Keep the release tag and source code (auto-generated by GitHub) intact.
- README install instructions should point to building from source; remove or soften any "download zip" guidance that implies a prebuilt binary.

### Settings Window Cleanup

Remove these controls and their localized strings from `SettingsView`:

- `sensitivity` slider
- `normal_step` slider
- `fine_step` slider
- `fine_control` toggle
- `continuous_window` slider
- `cursor_freeze` toggle
- `bottom_quarter_only` toggle

Keep these controls:

- Enable app
- Launch at login
- Volume edge gesture toggle
- Brightness edge gesture toggle
- Swap sides toggle
- Edge width slider
- Middle click toggle
- Smart typing detection toggle
- Typing cooldown slider
- Permissions status section (Accessibility, Input Monitoring, Can listen, open settings buttons, refresh)

### Menu Bar Dropdown

- Remove the `Permissions…` menu item from `MenuBarController`.
- Remove the `openPermissions` action and the `permissionManager` dependency from `MenuBarController` if it becomes unused.
- Keep `Settings…`, `Debug…`, enable/disable, and `Quit`.

### Cursor Behavior

- Remove `freezeCursor(at:)` and `unfreezeCursor()` from `SystemControlling` protocol and `SystemControl`.
- Remove `frozenPosition`, `CGWarpMouseCursorPosition`, `CGAssociateMouseAndMouseCursorPosition`, and `NSCursor.hide()/unhide()` calls.
- Remove `cursorFreeze` from `FeatureToggles` and all default/test references.
- Ensure `adjustVolume` and `adjustBrightness` never touch the cursor — they already only post media keys, so this is verified by removal of the only cursor-moving code.

## Settings Model Compatibility

- Removed setting fields (`sensitivity`, `normalStep`, `fineStep`, `fineControl`, `continuousWindowSeconds`, `cursorFreeze`, `bottomQuarterOnly`) must remain backward-compatible on decode: existing saved settings files with these keys should not crash. `Codable` synthesized decoding ignores unknown keys, and `decodeIfPresent` provides defaults for removed required keys, so no explicit migration is needed beyond removing the fields from the struct.
- `ActionDispatcher.signedDelta` currently reads `fineControl`, `fineStep`, `normalStep`, and `sensitivity`. After removing these fields, `signedDelta` must use a fixed step of `1.0` so each emitted gesture maps to exactly one media-key press. This matches the discrete step model already implemented in `GestureRecognizer`.

## Components

### `SettingsView.swift`

- Remove the seven controls listed above.
- Keep the remaining controls and layout.

### `MenuBarController.swift`

- Remove `permissionManager`, `openPermissions`, and the `Permissions…` menu item.

### `SystemControl.swift`

- Remove `freezeCursor`, `unfreezeCursor`, `frozenPosition`, and all CoreGraphics cursor calls.

### `AppSettings.swift`

- Remove `cursorFreeze`, `bottomQuarterOnly`, `fineControl` from `FeatureToggles`.
- Remove `sensitivity`, `normalStep`, `fineStep`, `continuousWindowSeconds` from `GestureSettings`.
- Keep `edgeWidthPercent`, `typingCooldownSeconds`, `physicalStepDistance`, `physicalStepIntervalSeconds`.
- Update `default` and `validated()` accordingly.

### `ActionDispatcher.swift`

- Simplify `signedDelta` to return `sign * magnitude` with a fixed multiplier so one step = one media-key press. Since `magnitude` is already `1.0` from the recognizer, the delta passed to `SystemControl` will be `+1.0` or `-1.0`.

### `GestureRecognizer.swift`

- Remove references to `bottomQuarterOnly` filtering (the toggle is removed). Physical edge recognition remains.
- Remove references to `smartTypingDetection` only if its toggle is also removed — it is kept, so no change needed there.

### `Localizable.strings` (en + zh-Hans)

- Remove keys for removed controls.
- Keep keys for remaining controls and menu items.

### `SlidrFreeCoreChecks/main.swift`

- Update default-settings checks to no longer assert removed fields.
- Update validation-clamp checks to no longer reference removed fields.
- Update migration test to not reference removed fields.

### `README.md` / `README.zh-CN.md`

- Update install instructions to reflect source-only release.

### `scripts/package-release.sh`

- Keep the script for local development builds, but it no longer needs to run during release publishing.

## Error Handling

- No new error paths. Media-key posting failure already returns `.failed`.
- Removing cursor code eliminates the risk of stuck hidden cursors.

## Acceptance Criteria

- `gh release view v0.1.0` shows no binary assets, only source code.
- Settings window has no sliders/toggles for sensitivity, normal step, fine step, fine control, continuous window, cursor freeze, or bottom quarter only.
- Menu bar dropdown has no `Permissions…` item.
- `grep` for `freezeCursor`, `CGWarpMouseCursorPosition`, `cursorFreeze` in `Sources/` returns no matches.
- `swift test` and `swift run SlidrFreeCoreChecks` pass.
- `swift build` succeeds.
- No cursor movement occurs during volume/brightness gestures (verified by code removal).
