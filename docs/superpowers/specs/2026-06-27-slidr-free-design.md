# Slidr-Free 开源替代品设计规格

日期：2026-06-27

## 1. 项目定位

Slidr-Free 是一个独立的免费开源 macOS 触控板增强工具。项目参考同类工具公开宣传的功能理念，但不复制任何第三方产品的名称、品牌、图标、文案、截图、视觉风格、私有实现或二进制行为细节。

README 和发布页必须明确说明：本项目是独立开源项目，与 Slidr 或其他商业产品没有官方关联。

## 2. 首版范围

目标平台：macOS 13+

技术路线：Swift + AppKit/SwiftUI 原生 macOS 菜单栏应用。

分发方式：源码 + 未签名 Release 包。README 说明自行构建方式、Gatekeeper 提示和右键打开方式。

许可证：MIT。

首版包含以下功能，并且每项功能都必须能独立启用或禁用：

- 菜单栏常驻 App
- 左/右触控板边缘纵向滑动
- 边缘滑动调音量
- 边缘滑动调内置屏幕亮度
- 三指中键点击
- 精细调节模式
- 左右边缘互换
- 底部四分之一区域模式
- 打字时忽略手势
- 调节时冻结光标
- 登录时启动
- 权限状态检查与引导

首版不支持外接显示器 DDC/CI 亮度调节。

## 3. 默认交互行为

默认映射：

- 左边缘纵向滑动：调节内置屏幕亮度
- 右边缘纵向滑动：调节音量
- 向上滑动：增加
- 向下滑动：降低

默认识别策略：

- 边缘区域宽度约为触控板左右各 8%–12%。
- 手势触发后进入短时间连续调节窗口，避免每个微小移动都触发一次系统动作。
- 精细模式开启后降低步进，例如普通模式每次约 1 格，精细模式约 0.25–0.5 格。
- 底部四分之一区域模式开启后，只有手指位于触控板下方区域时才识别边缘滑动。
- 左右互换开启后，左边缘调音量，右边缘调亮度。
- 打字检测开启后，最近约 0.7–1.2 秒内有键盘输入时暂停手势识别。
- 光标冻结开启后，手势调节期间保持光标位置不漂移，手势结束后恢复正常。
- 三指中键点击开启后，识别三指轻点/点击并发送中键点击事件。

所有默认数值都应集中定义，后续可暴露为设置项。

## 5. 技术架构

系统按“监听、识别、动作执行”三层分离，避免输入事件、手势状态机和系统控制耦合。

### AppEntry

负责应用启动、依赖初始化、设置加载、权限检查、输入监听启动和菜单栏状态项创建。

### MenuBarController

基于 `NSStatusItem` 实现菜单栏入口。提供快速启停、打开设置、查看权限状态和退出 App。

### SettingsView

SwiftUI 设置窗口。所有功能开关和参数写入 `UserDefaults`。设置变更应即时通知监听和识别模块。

### PermissionManager

负责检查和引导 Accessibility / Input Monitoring 权限。

可使用：

- `AXIsProcessTrustedWithOptions`
- `kAXTrustedCheckOptionPrompt`
- `CGPreflightListenEventAccess()`

未授权时禁用核心监听并显示明确提示，不允许静默失败。

### InputEventTap

基于 `CGEventTap` 全局监听鼠标、滚动、点击和键盘事件。优先采用 listen-only event tap。该模块只输出标准化输入事件，不执行业务动作。

### GestureRecognizer

负责把标准化输入事件转换为语义手势事件，包括：

- 左边缘滑动
- 右边缘滑动
- 三指中键点击候选
- 打字冷却抑制
- 底部四分之一区域过滤
- 光标冻结触发窗口

该模块不直接调节音量、亮度或注入鼠标事件。

### ActionDispatcher

把语义手势事件映射到系统动作：

- 音量增加/降低
- 亮度增加/降低
- 中键点击
- HUD 或用户反馈

### SystemControl

封装所有系统交互，包括音量、内置屏幕亮度、鼠标事件注入、光标位置管理和 HUD/反馈。由于部分 macOS 系统控制能力可能受权限或 API 稳定性影响，该层必须保持可替换。

## 6. 权限与系统能力

应用需要处理以下权限：

- Accessibility：用于辅助功能相关控制和可能的事件注入。
- Input Monitoring：用于全局输入监听。

用户体验要求：

- 首次启动显示权限状态。
- 权限缺失时说明影响范围。
- 提供打开系统设置的入口。
- 用户拒绝权限时应用不崩溃，菜单栏和设置仍可使用。

## 7. 技术风险与降级策略

### 触控板原始手势数据限制

macOS 不一定向普通 App 完整暴露触控板多指和物理边缘信息。边缘滑动和三指点击可能需要通过事件特征近似判断。因此首版必须保证：

- 每项功能可关闭。
- 灵敏度和边缘宽度可调。
- README 标注实验性限制。

### HUD 与系统控制限制

音量控制可优先通过系统音量 API 或模拟媒体键实现。内置屏幕亮度可通过 CoreDisplay / IOKit 相关能力封装实现。

如果系统原生 HUD 无法稳定调用，首版可使用独立轻量反馈 UI，但不能模仿任何第三方产品视觉。

### 未签名发布限制

未签名 Release 包可能触发 Gatekeeper 提示。README 必须说明：

- 下载版为未签名构建。
- 用户可自行从源码构建。
- 如选择运行下载版，可能需要右键打开或在系统设置中允许。

- 功能关闭后绝不触发对应动作。

- 退出 App 后 event tap 被清理。
- 设置变更无需重启即可生效。
- 睡眠/唤醒后监听可恢复或给出明确错误状态。

## 11. Clean-room 要求

开发过程中不得：

- 反编译或复制第三方商业软件代码。
- 复制第三方软件图标、截图、文案、名称或视觉风格。
- 在项目名、App 名或图标中使用可能造成混淆的元素。

允许：

- 参考公开网页中描述的功能类别。
- 使用 Apple 官方文档和公开 API。
- 根据用户需求独立设计交互、UI 和实现。
