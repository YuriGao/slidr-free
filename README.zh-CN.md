# Slidr Free

[English](README.md) | 简体中文

Slidr Free 是一款开源 macOS 菜单栏工具，提供物理触控板边缘手势和可选的三指中键点击测试功能。

本项目是独立开源项目，与任何同名商业产品或供应商无关。

## 系统要求

- macOS 13 或更高版本
- Swift 5.9 或更高版本

## 功能

- **左右边缘滑动** — 沿物理触控板左侧或右侧边缘上下滑动，用于调节亮度和音量。默认左侧调节亮度，右侧调节音量。
- **左右互换** — 可在设置中交换左右边缘对应的动作。
- **上边缘浏览器标签页滑动** — 沿物理触控板上边缘左右滑动，在 Safari、Google Chrome 和 Microsoft Edge 中切换标签页，并在每次切换时提供触感反馈。
- **三指中键点击（测试功能）** — 恰好使用三根手指，通过轻点或物理按下来触发中键点击。

## 三指中键点击测试功能

中键点击**默认关闭**。请从菜单栏项目进入 **设置… → 中键点击** 后启用。该手势固定为恰好三指；此测试版不提供手指数、移动阈值或时长阈值调节。

- **轻点**：符合条件的三指放置释放后，发送一组中键按下/抬起事件。单独的“启用三指轻点”偏好默认开启，但只在中键点击总开关启用时生效。
- **物理按下**：新鲜的三指和弦处于活动状态时，把匹配的左键或右键按下/拖动/抬起事件转换为完整配对的中键事件。同一次触摸会话中，轻点和物理按下互斥。
- 设置界面会显示触控监视器和物理点击 Event Tap 的有限运行状态。发生降级时，普通鼠标输入会原样放行，不会猜测或转换。

如果 macOS 的**三指拖移**或**查询与数据检测器**占用相同手势，请在系统触控板设置中停用或更改对应手势，避免冲突。

## 实验性物理触控板支持

由于 macOS 没有提供获取每根手指物理触控板坐标的公开 API，物理触控板手势通过 Apple 私有 `MultitouchSupport` 框架读取触摸帧。

- 目前没有公开 API 或屏幕边缘光标回退方案。私有框架不可用或发生变化时，触控输入管线会停用，并在设置中显示状态。
- Slidr Free 只打开 `MTDeviceCreateDefault` 返回的默认物理触控设备，不会枚举、分类或路由多个设备，因此无法保证系统选择的是内置还是外接触控板。
- 物理按下转换使用全局鼠标 Event Tap。macOS 无法为这些左右键事件可靠提供来源设备身份；三指和弦处于有效状态时点击另一只鼠标，该事件可能被转换为中键点击。
- 私有 API 在不同硬件或 macOS 版本上可能变化。符号缺失、设备不可用、权限丢失或无触摸帧时，功能应降级且不导致应用崩溃。

## 设置与权限

菜单栏的**设置…**窗口提供边缘手势、中键点击、登录时启动和运行状态控制。

Slidr Free 需要**辅助功能（Accessibility）**权限，以监听物理手势使用的全局输入流。请在**系统设置 → 隐私与安全性 → 辅助功能**中授予。替换或重新构建临时签名的应用后，macOS 可能保留旧的 TCC 记录；如状态异常，请退出 Slidr Free，在辅助功能列表中移除并重新添加当前应用，然后重启并刷新状态。

## 构建、测试和打包

```bash
git clone https://github.com/YuriGao/slidr-free.git
cd slidr-free

swift run SlidrFreeCoreChecks
swift test
swift build
bash scripts/package-release.sh
bash scripts/verify-release.sh
```

打包会生成 `release/Slidr-Free.app` 和 `release/Slidr-Free.app.zip`。验证脚本会检查包标识、版本 `0.3.0` / 构建号 `3001`、包内 MIT 许可证、压缩包内许可证和临时签名。应用设置了 `LSUIElement=true`，因此不显示 Dock 图标，并通过菜单栏项目控制。临时签名不等于 Developer ID 签名或公证。

## 已知限制

- 中键点击只支持默认物理触控设备上的恰好三指手势，不包含可配置手指数、多设备路由或按应用排除。
- 受上述全局鼠标来源限制影响，三指和弦有效时，外接鼠标点击可能被转换。
- 上边缘标签页切换只在 Safari、Google Chrome 或 Microsoft Edge 位于前台时执行。
- 自动化测试不依赖真实辅助功能/TCC 权限或物理硬件，硬件行为仍需执行批准设计中的手工测试矩阵。

## 回滚

已知正常基线是标签 `v0.2.0`，对应提交 `eb93e18e9ba225502bac580ae98e006c1bf1aec5`。替换已安装的 v0.2.0 前，请保存其安装包、记录该安装包的 SHA-256，并备份 `SlidrFree.settings.v1` 偏好数据。回滚时退出 Slidr Free、恢复 v0.2.0 应用包，仅在 macOS 无法识别时重置辅助功能权限，然后重新启动并核对显示版本及全部已有边缘手势。正式公开发布前，还应演练完整的 v0.3 → v0.2 → v0.3 设置兼容路径。
