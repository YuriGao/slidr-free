# Slidr Free

[English](README.md) | 简体中文

Slidr Free 是一款开源 macOS 菜单栏工具，提供物理触控板边缘手势和可配置的中键点击测试功能。

本项目是独立开源项目，与任何同名商业产品或供应商无关。

## 系统要求

- macOS 13 或更高版本
- Swift 5.9 或更高版本

## 功能

- **左右边缘滑动** — 沿物理触控板左侧或右侧边缘上下滑动，用于调节亮度和音量。默认左侧调节亮度，右侧调节音量。
- **直接边缘分配** — 左右两侧可分别选择亮度、音量或无；上边缘可选择浏览器标签页或无。
- **独立触发距离** — 左边缘、右边缘和上边缘可分别设置每次触发所需的物理移动距离，数值越小越灵敏。
- **上边缘浏览器标签页滑动** — 沿物理触控板上边缘左右滑动，在 Safari、Google Chrome 和 Microsoft Edge 中切换标签页，并在每次切换时提供触感反馈。
- **边缘起点限制** — 单次触控必须从设定的边缘宽度范围内开始；从其他区域滑入边缘不会触发，直到所有手指抬起后才重新判断。
- **可配置中键点击（测试功能）** — 恰好使用 2、3 或 4 根手指，通过轻点或物理按下来触发中键点击，默认使用 4 指。
- **首次引导与安全测试** — 首次运行检查兼容性和权限，并在不派发系统动作的情况下验证手势。
- **统一健康状态与诊断** — 菜单栏和概览使用同一可操作状态；复制诊断前可预览不含敏感信息的摘要。

## 可配置中键点击测试功能

中键点击**默认关闭**。请从菜单栏项目进入 **设置… → 中键点击** 后启用，并选择 2、3 或 4 指。新安装和旧配置迁移后均默认使用恰好 4 指。轻点与物理按下共用这一设置；移动和时长阈值在此测试版中仍为固定值。

- **轻点**：符合所选精确手指数的放置释放后，发送一组中键按下/抬起事件。单独的“启用轻点”偏好默认开启，但只在中键点击总开关启用时生效。
- **物理按下**：新鲜的精确手指数和弦处于活动状态时，把匹配的左键或右键按下/拖动/抬起事件转换为完整配对的中键事件。同一次触摸会话中，轻点和物理按下互斥。
- 设置界面会显示触控监视器和物理点击 Event Tap 的有限运行状态。发生降级时，普通鼠标输入会原样放行，不会猜测或转换。

启用 macOS **三指拖移**时请选择 4 指。2 指选项可能与 macOS 辅助点按和常用双指手势冲突。

## 实验性物理触控板支持

由于 macOS 没有提供获取每根手指物理触控板坐标的公开 API，物理触控板手势通过 Apple 私有 `MultitouchSupport` 框架读取触摸帧。

- 目前没有公开 API 或屏幕边缘光标回退方案。私有框架不可用或发生变化时，触控输入管线会停用，并在设置中显示状态。
- Slidr Free 只打开 `MTDeviceCreateDefault` 返回的默认物理触控设备，不会枚举、分类或路由多个设备，因此无法保证系统选择的是内置还是外接触控板。
- 物理按下转换使用全局鼠标 Event Tap。macOS 无法为这些左右键事件可靠提供来源设备身份；有效的中键手势和弦处于活动状态时点击另一只鼠标，该事件可能被转换为中键点击。
- 私有 API 在不同硬件或 macOS 版本上可能变化。符号缺失、设备不可用、权限丢失或无触摸帧时，功能应降级且不导致应用崩溃。

## 设置与权限

菜单栏的**设置…**窗口包含概览、边缘手势、中键点击和诊断四个入口。新安装在完成设置向导前不会启用真实动作；v0.3 的边缘行为会无损迁移为直接分配。

Slidr Free 需要**辅助功能（Accessibility）**权限，以监听物理手势使用的全局输入流。请在**系统设置 → 隐私与安全性 → 辅助功能**中授予。替换或重新构建临时签名的应用后，macOS 可能保留旧的 TCC 记录；如状态异常，请退出 Slidr Free，在辅助功能列表中移除并重新添加当前应用，然后重启并刷新状态。

## 构建、测试和打包

```bash
git clone https://github.com/YuriGao/slidr-free.git
cd slidr-free

swift run SlidrFreeCoreChecks
swift test
swift build
bash scripts/package-release.sh
```

打包会生成 `release/Slidr-Free.app` 和 `release/Slidr-Free.app.zip`，并验证版本 `0.4.0` / 构建号 `4000`、包标识、许可证、压缩包内容和明确的 ad-hoc 临时签名。仓库当前不发布 Developer ID 签名或 Apple 公证的二进制包；该制品仅用于本地开发和测试，运行时可能需要确认 macOS 安全提示，重新构建后也可能需要重新授予辅助功能权限。详见 [v0.4 轻量验收清单](docs/qa/v0.4-release-acceptance.md)。

## 已知限制

- 中键点击只支持默认物理触控设备上的精确 2–4 指手势，不包含更高指数组合、“至少 N 指”匹配、多设备路由或按应用排除。
- 受上述全局鼠标来源限制影响，中键和弦有效时，外接鼠标点击可能被转换。
- 上边缘标签页切换只在 Safari、Google Chrome 或 Microsoft Edge 位于前台时执行。
- 自动化测试不依赖真实辅助功能/TCC 权限或物理硬件；其他 Mac 上的内置和外接触控板验证属于非阻塞的社区测试，不作为 v0.4 放行门槛。

## 回滚

已知正常基线是标签 `v0.2.0`，对应提交 `eb93e18e9ba225502bac580ae98e006c1bf1aec5`。替换已安装的 v0.2.0 前，请保存其安装包、记录该安装包的 SHA-256，并备份 `SlidrFree.settings.v1` 偏好数据。回滚时退出 Slidr Free、恢复 v0.2.0 应用包，仅在 macOS 无法识别时重置辅助功能权限，然后重新启动并核对显示版本及全部已有边缘手势。
